# AGENTS.md

이 저장소는 **'게임 기획자 Level Up!'** 책 집필을 위한 AI 에이전트들의 작업 지침서입니다.
모든 에이전트는 아래의 필수 규칙을 준수하여 원고를 작성하고 협업해야 합니다.

## 1. 책의 목적과 방향성

- **대상 독자**: 게임 기획자 지망생, 주니어 기획자, 연관 직군 실무자
- **핵심 목표**: 단순 이론서가 아닌, 실제 현업 단위로 사고하고 운영 단계까지 구조적으로 고려할 수 있는 **기획 입문자**가 **실무형 게임 기획자**로 성장하도록 돕는 것이다.
- **공통 경험 목표**:
  - 독자는 책을 따라가며 기획 기초, 시스템 설계, 성장/경제, 협업, 출시, 운영까지 단계적으로 익혀야 한다.
  - 책 전체는 하나의 **가상 모바일 수집형 RPG 프로젝트**를 함께 만드는 구조여야 한다.
    - 프로젝트명: 택틱스 오브 제로
    - 장르: 모바일 수집형 RPG
    - 기본 전제: 성역들의 쾌락에 의해 타락한 세계 Nova-3K에서, 영웅을 수집하고 조합해 부패한 질서를 무너뜨리고 세계를 리셋하는 턴제 RPG를 만든다.
    - 독자 목표: 재미를 말하는 단계에서 멈추지 않고, 출시와 운영까지 고려해 구조를 설계하는 기획자로 성장한다.
  - 모든 장은 아래 두 가지를 동시에 전진시켜야 한다.
    - 독자의 기획 실무 역량
    - 프로젝트의 구체적 산출물

## 2. 에이전트 집필 원칙 (톤앤매너)

- **문체**: **평어체(~한다, ~이다)**를 기본으로 하며, 존댓말과 불필요한 수식어를 뺍니다.
- **실무 중심**: '정답'처럼 단정하지 않습니다. 항상 조건, 이유, 한계, 그리고 실제 현업 맥락(개발/운영/사업)을 함께 씁니다.
- **경험 기반**: 기획자는 문서 작성자가 아닌 문제 해결자입니다. 플레이어 경험과 개발/서비스 현실의 균형을 잡는 과정을 보여줍니다.
- **사실 검증**: 확인되지 않은 뇌피셜(추측성 수치, 특정 게임 비공개 지표 등)을 사실처럼 쓰지 않습니다.
- **연속성 유지**: 앞 장에서 만든 모바일 수집형 RPG의 설정과 판단을 다음 장에서 함부로 초기화하지 않습니다. 변경이 필요하면 왜 바뀌는지, 어떤 비용과 이득이 있는지 함께 설명합니다.
- **장르 맥락 유지**: 설명 예시는 가능한 한 모바일 수집형 RPG의 개발, 운영, 사업 현실 안에서 풀어냅니다. 다른 장르 예시는 비교가 꼭 필요할 때만 보조적으로 사용합니다.

## 3. 원고 구조 및 템플릿

각 퀘스트(Quest/장)는 아래 순서로 구성합니다.

1. **퀘스트 목표**
   - 이 퀘스트에서 달성해야 할 독자의 기획 핵심 역량과 프로젝트(택틱스 오브 제로)의 구체적인 산출물을 1~2줄로 명확히 정의한다.

2. **왜 이 퀘스트가 필요한가 (문제 제기)**
   - 현업에서 실제로 마주칠 수 있는 구체적인 문제 상황을 제시하여 퀘스트의 필요성을 체감하게 한다.
   - 개발, 운영, 사업적 맥락에서 발생할 수 있는 딜레마나 어려움을 묘사한다.

3. **클리어에 필요한 개념**
   - 문제 해결을 위해 알아야 할 기획 지식, 시스템 구조, 또는 프레임워크를 설명한다.
   - 단순 이론 나열을 피하고, 실제 판단 과정에서 이 개념이 어떻게 기준이 되는지 실무 중심으로 서술한다.

4. **훈련과 실무적용**
   - 앞서 배운 개념을 가상의 모바일 수집형 RPG 프로젝트에 직접 적용해 결과물을 도출한다.
   - 단정적인 '정답'을 제시하기보다, 여러 선택지 중 왜 이 방식을 택했는지(조건, 이유, 한계) 판단 과정을 함께 보여준다.

5. **퀘스트 클리어 Level Up!**
   - 이번 퀘스트에서 얻은 기획적 통찰과 산출물을 요약한다.
   - 여기서 내린 결정이 다음 퀘스트(개발/운영의 다음 단계)와 어떻게 연결되는지 명시하여 프로젝트의 연속성을 부여한다.

### 권장 흐름 요약

`프로젝트 상황 제시 -> 목표 제시 -> 문제 체감 -> 개념 습득 -> 훈련과 실무 적용 -> 클리어 확인 -> 다음 개발/운영 단계 연결`

### 집필 체크포인트

- 이 퀘스트의 문제 상황이 충분히 구체적인가?
- 개념 설명이 실제 판단 기준으로 이어지는가?
- 훈련과 실무적용 파트가 앞의 개념을 실제 판단과 산출물로 회수하는가?
- 이번 퀘스트가 책 전체의 모바일 수집형 RPG 프로젝트를 실제로 한 단계 전진시키는가?

## 5. 디렉토리 구조 및 파이프라인

한 번에 크게 쓰지 않고, 단위를 쪼개어 다음 디렉토리에 누적합니다. **작업 전, 다른 에이전트의 완료된 산출물을 반드시 먼저 확인(Context)** 해야 합니다.

- `index.md`: 전체 목차 문서
- `/quests`: 실제 책 본문 원고 (파일명 예시: `quest_01.md`)
