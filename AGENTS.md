# AGENTS.md

이 저장소는 **'게임 기획자 Level Up!'** 책 집필을 위한 AI 에이전트들의 작업 지침서입니다.
모든 에이전트는 아래의 필수 규칙을 준수하여 원고를 작성하고 협업해야 합니다.

## 1. 책의 목적과 방향성

- **대상 독자**: 게임 기획자 지망생, 주니어 기획자, 연관 직군 실무자
- **핵심 목표**: 단순 이론서가 아닌, 실제 현업 단위로 사고하고 운영 단계까지 구조적으로 고려할 수 있는 **기획 입문자**가 **실무형 게임 기획자**로 성장하도록 돕는 것이다.
- **공통 경험 목표**:
  - 독자는 책을 따라가며 기획 기초, 시스템 설계, 성장/경제, 협업, 출시, 운영까지 단계적으로 익혀야 한다.
  - 책 전체는 하나의 **가상 모바일 수집형 RPG 프로젝트**를 함께 만드는 구조여야 한다.
  - 모든 장은 아래 두 가지를 동시에 전진시켜야 한다.
    - 독자의 기획 실무 역량
    - 프로젝트의 구체적 산출물

## 2. 에이전트 집필 원칙 (톤앤매너)

- **문체**: **평어체(~한다, ~이다)**를 기본으로 하며, 존댓말과 불필요한 수식어를 뺍니다.
- **실무 중심**: '정답'처럼 단정하지 않습니다. 항상 조건, 이유, 한계, 그리고 실제 현업 맥락(개발/운영/사업)을 함께 씁니다.
- **경험 기반**: 기획자는 문서 작성자가 아닌 문제 해결자입니다. 플레이어 경험과 개발/서비스 현실의 균형을 잡는 과정을 보여줍니다.
- **사실 검증**: 확인되지 않은 뇌피셜(추측성 수치, 특정 게임 비공개 지표 등)을 사실처럼 쓰지 않습니다.
- **연속성 유지**: 앞 장에서 만든 모바일 수집형 RPG의 설정과 판단을 다음 장에서 함부로 초기화하지 않습니다. 변경이 필요하면 왜 바뀌는지, 어떤 비용과 이득이 있는지 함께 설명합니다.
- **장르 맥락 유지**: 설명 예시는 가능한 한 모바일 수집형 RPG의 개발, 운영, 사업 현실 안에서 풀어냅니다. 다른 장르 예시는 비교가 꼭 필요할 때만 보조적으로 사용합니다.

## 3. 원고 구조 및 템플릿

각 퀘스트(Quest/장)는 아래 순서로 구성합니다.

1. **퀘스트 목표**
   - 이 장을 끝냈을 때 독자가 현재 진행 중인 모바일 수집형 RPG 프로젝트에서 무엇을 할 수 있어야 하는지 명확히 쓴다.

2. **왜 이 퀘스트가 필요한가 (문제 제기)**
   - 초보자가 자주 빠지는 오해, 현업에서 실제로 부딪히는 문제, 잘못 판단했을 때 생기는 결과를 **현재 프로젝트 상황**과 함께 먼저 보여준다.
   - 독자가 "그래서 이걸 배워야 하는 이유"를, 우리가 만들고 있는 RPG가 어떻게 흔들리는지를 통해 체감하게 만든다.

3. **클리어에 필요한 개념**
   - 문제를 풀기 위해 꼭 알아야 할 용어, 구조, 판단 기준을 설명한다.
   - 개념은 정의에서 끝내지 않는다. "이 개념을 왜 쓰는가", "모바일 수집형 RPG의 어떤 선택을 돕는가"까지 함께 쓴다.
   - 여러 개념이 나오면 나열하지 말고, 실제 판단 순서대로 배치한다.

4. **훈련**
   - 개념을 외우는 대신, 같은 RPG 프로젝트 안에서 어떻게 판단해야 하는지 보여준다.
   - 비교 질문, 체크리스트, 짧은 사고 실험, 잘못된 예와 수정 예를 활용할 수 있다.
   - 독자가 "이론을 이해했다"가 아니라 "이 게임의 다음 결정을 이 기준으로 생각해 볼 수 있다" 상태가 되게 만든다.

5. **실무 적용**
   - 앞에서 배운 기준을 실제 업무 맥락에 연결한다.
   - 사례는 원칙적으로 **책 전체의 공통 모바일 수집형 RPG 프로젝트**를 사용한다. 필요하다면 특정 기능 개선, 협업 상황, 운영 이슈 대응처럼 복잡도를 통제해 보여준다.
   - "이론 설명"이 아니라 "그래서 이 게임을 만드는 팀은 지금 이렇게 판단하고 정리한다"를 보여준다.

6. **퀘스트 클리어 Level Up!**
   - 이 장의 핵심 내용들을 짧게 정리한다.
   - 독자가 스스로 점검해볼 만한 질문이나 다음 퀘스트로 이어지는 연결 질문을 둔다.
   - 끝맺음은 요약이 아니라, 독자의 사고 수준이 한 단계 올라갔는지, 그리고 **이 모바일 수집형 RPG 프로젝트가 다음 단계로 넘어갈 준비가 되었는지** 확인하는 장치가 되어야 한다.

### 권장 흐름 요약

`프로젝트 상황 제시 -> 목표 제시 -> 문제 체감 -> 개념 습득 -> 판단 훈련 -> 같은 게임에 적용 -> 클리어 확인 -> 다음 개발/운영 단계 연결`

### 집필 체크포인트

- 이 퀘스트의 문제 상황이 충분히 구체적인가?
- 개념 설명이 실제 판단 기준으로 이어지는가?
- 독자가 직접 생각하거나 써보는 단계가 들어 있는가?
- 실무 적용 사례가 앞의 개념과 연습을 실제로 회수하는가?
- 이번 퀘스트가 책 전체의 모바일 수집형 RPG 프로젝트를 실제로 한 단계 전진시키는가?
- 퀘스트를 끝냈을 때 "그래서 이제 이 게임의 무엇을 할 수 있는가"가 분명한가?

## 5. 디렉토리 구조 및 파이프라인

한 번에 크게 쓰지 않고, 단위를 쪼개어 다음 디렉토리에 누적합니다. **작업 전, 다른 에이전트의 완료된 산출물을 반드시 먼저 확인(Context)** 해야 합니다.

- `index.md`: 전체 목차 문서
- `/quests`: 실제 책 본문 원고 (파일명 예시: `quest_01.md`)
